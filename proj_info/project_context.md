# Общий контекст проекта SZI Knapsack Solver

Данный файл содержит краткую сводку для LLM, чтобы быстро войти в курс дела.

## Назначение
Решение задачи выбора набора СЗИ для защиты группы информационных активов (ГИА) при заданных бюджетных ограничениях.

## Основные сущности
- **СЗИ (Средство защиты информации):** имеет два ключевых параметра (время взлома `c` и время реакции `d`).
- **ГИА (Группа информационных активов):** набор активов, для которых действуют общие ограничения бюджета.

## Краткое описание файлов
- `app.py`: FastAPI бэкенд (порт 8000). Маркирует `/static` и предоставляет `/api/solve`.
- `solver.py`: Алгоритм решения. Использует `pulp` (линейное программирование).
- `main_window.py`: Лаунчер для десктоп-версии (pywebview).
- `static/index.html`: Главный интерфейс (три вкладки: Оптимизация, Документация, Настройки).
- `static/script.js`: Логика фронтенда (Fetch API, построение таблиц, переход между темами).
- `static/style.css`: Описание стилей и темная/светлая темы (CSS Variables).
- `requirements.txt`: Список всех зависимостей.
- `railway.toml`, `Procfile`: Конфиги для облачного деплоя.
- `build_exe.bat`, `main_window.spec`: Сборка под Windows (Portable EXE).

## Логика решения
Алгоритм является гибридным:
1. Решает аддитивную задачу о рюкзаке (через PuLP).
2. Генерирует варианты на основе максиминного критерия (время реакции самого слабого звена).
3. Объединяет их через весовой коэффициент λ.

Вход (JSON): `m, n, lam, A, b, c, d`
Выход (JSON): `vector_x, optimal_f, recommended_szi`
