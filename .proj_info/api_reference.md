# API Reference

Бэкенд предоставляет следующие эндпоинты.

## `GET /`
Возвращает основную HTML-страницу (`static/index.html`).

## `POST /api/solve`
Принимает JSON с параметрами задачи и возвращает найденное решение.

### Параметры тела запроса (SolveRequest):
- `m` (int): Количество ограничений (групп активов).
- `n` (int): Количество доступных СЗИ.
- `lam` (float): Коэффициент λ ∈ [0, 1] для взвешивания критериев.
- `A` (List[List[float]]): Матрица стоимости (m × n).
- `b` (List[float]): Вектор финансовых ограничений (длина m).
- `c` (List[float]): Индивидуальное время обхода для каждого СЗИ (длина n).
- `d` (List[float]): Минимальное время реагирования отдела безопасности для каждого СЗИ (длина n).

### Ответ:
- `success` (bool): Флаг успешного завершения.
- `vector_x` (List[int]): Бинарный вектор выбранных СЗИ (длина n) для наилучшего решения.
- `optimal_f` (float): Значение комбинированной целевой функции наилучшего решения.
- `recommended_szi` (List[int]): Список порядковых номеров (с 1) выбранных СЗИ лучшего решения.
- `all_solutions` (List[Dict]): Список всех найденных векторов кандидатов (x_0, x_1, ... x_{n-1}) с их параметрами.
- `error` (str, опционально): Описание ошибки при `success: false`.
