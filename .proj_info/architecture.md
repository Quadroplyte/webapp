# Архитектура проекта

Проект построен по классической клиент-серверной архитектуре с возможностью упаковки в единый исполняемый файл.

## Структура каталогов
- `app.py`: Основной файл FastAPI приложения. Служит точкой входа для бэкенда.
- `solver.py`: Ядро проекта — реализация алгоритмов оптимизации.
- `main_window.py`: Код для создания настольного окна (pywebview), которое запускает FastAPI внутри себя.
- `static/`: Папка с фронтендом (index.html, style.css, script.js, lang.js).
- `proj_info/`: Контекстная информация о проекте (данная папка).
- `requirements.txt`: Зависимости для разработки и десктопной версии.
- `requirements-server.txt`: Зависимости для серверного деплоя (без тяжелых библиотек для графики).

## Взаимодействие
1. Фронтенд (браузер или pywebview) отправляет POST-запрос на `/api/solve` с JSON-данными.
2. `app.py` получает запрос, валидирует его с помощью Pydantic и передаёт в `solver.py`.
3. `solver.py` использует библиотеку `pulp` для нахождения оптимального решения.
4. Результат возвращается в JSON-формате и отображается на фронтенде.
