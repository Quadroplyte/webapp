# Архитектура проекта

Проект построен по классической клиент-серверной архитектуре с возможностью упаковки в единый исполняемый файл.

## Структура каталогов
- `app.py`: Основной файл FastAPI приложения. Служит точкой входа для бэкенда.
- `solver.py`: Ядро проекта — реализация алгоритмов оптимизации.
- `main_window.py`: Код для создания настольного окна (pywebview), которое запускает FastAPI внутри себя.
- `static/`: Папка с фронтендом (index.html, style.css, script.js, lang.js).
- `.proj_info/`: Контекстная информация о проекте (данная папка).
- `requirements.txt`: Зависимости для разработки и десктопной версии.
- `requirements-server.txt`: Зависимости для серверного деплоя (без тяжелых библиотек для графики).

## Взаимодействие
1. Фронтенд (браузер или pywebview) отправляет POST-запрос на `/api/solve` с JSON-данными.
2. `app.py` получает запрос, валидирует его с помощью Pydantic и передаёт в `solver.py`.
3. `solver.py` использует библиотеку `pulp` для нахождения оптимального решения.
4. Результат возвращается в JSON-формате и отображается на фронтенде.

## Система тем и стилизация
Проект использует современную систему управления темами на основе **CSS Variables** (кастомных свойств) и глобальную масштабируемую сетку.
- **Глобальное масштабирование**: Интерфейс увеличен на 30% для лучшей читаемости на десктопах (базовый размер `html` установлен в `97.5%`, `body` — `17px`).
- **Семантические токены**: Все цвета привязаны к функциональным именам (`--color-bg-base`, `--color-accent-main`, `--color-text-muted`).
- **Поддержка тем**: Темы переключаются через атрибут `data-theme` на элементе `<html>`.
- **Адаптивность**: Стили оптимизированы для десктопа и мобильных устройств. На экранах `<= 1024px` интерфейс перестраивается вертикально, а панель навигации фиксируется внизу.

### Глобальный фон
Фон задан через `--app-gradient` как `radial-gradient(ellipse at bottom right, ...)` и применён к `body` с `background-attachment: fixed`. Поверх наложен орнамент, выбираемый пользователем через атрибут `data-pattern` на `<html>`:
- `dots` — точечная сетка (CSS `radial-gradient`).
- `plus` — SVG плюсики на сетке 38×38px.
- `cross` — SVG крестики (×) на сетке 38×38px.
- `none` — чистый градиент без орнамента.

Цвета градиентов:
- **Светлая тема**: `#BFDBFE` (голубой) → `#FCE7F3` (розовый) → `#D1D3D9` (серый).
- **Тёмная тема**: `#2C200B` (янтарный) → `#101D30` (тёмно-синий) → `#141414` (чёрный).

### Акцентные цвета и Границы
- **Стиль границ (data-border-style)**: Проект внедряет систему управляемых акцентов через атрибуты `neutral`, `accent` или `none`.
  - **Neutral**: 1.8px серый/белый контур.
  - **Accent**: 2.3px розовый/золотой контур с мягким внешним свечением.
- **Светлая тема**: Акцентная обводка использует насыщенно-розовый цвет (`rgba(244, 114, 182, 0.8)`). 
- **Тёмная тема**: Акцентная обводка использует золотистый цвет (`rgba(250, 204, 21, 0.4)`).
- **Оптимальное решение**: Карточка лучшего решения (`.best-solution`) всегда имеет усиленную рамку **3.5px** и характерное свечение (`--best-card-shadow`), игнорируя общие настройки границ для сохранения визуального фокуса.
- **Инфо-панель (Cell Info)**: Отражает акцентные тона темы в своем фоне для визуальной консистентности.

### Glassmorphism и Типографика
Основные UI-компоненты используют усовершенствованный glassmorphism:
- **Масштабирование заголовков**: Основной заголовок (`.main-title`) в русской локализации — **32px**. Применяется эффект «объемного стекла» (3D Glass) с многослойными тенями. В светлой теме заголовок имеет темный стеклянный эффект с белыми бликами на гранях.
- **Glass Pill Nav** (`.glass-pill-nav`): полупрозрачные контейнеры с `backdrop-filter: blur(24px)` и улучшенной видимостью активных элементов.
- **Code Blocks / Модальные окна**: Используют эффект «выпуклого» (convex) стекла. Убраны внутренние (inset) тени для создания объема, направленного на пользователя. Темная тема модалок стала более прозрачной с усиленным блюром (32px).

### Режим производительности
При активации тумблера «Режим производительности» в настройках на `<html>` устанавливается атрибут `data-no-blur`. CSS-правила в `glass.css` под селектором `html[data-no-blur]` явно отключают `backdrop-filter` для каждого компонента и заменяют прозрачные фоны на сплошные (`--color-bg-surface`). Состояние сохраняется в `localStorage('perfMode')`.

## Структура фронтенда

### index.html
- **Шапка**: Заголовок приложения + два glass-pill (навигация, язык). На мобильных устройствах элементы управления прячутся в меню настроек, а панель опускается вниз.
- **Оптимизация**: Форма ввода, конфигурация параметров, матрицы, кнопки расчёта (единый вид в каждой теме), отображение результатов. На мобилках колонки становятся вертикальными.
- **Справка / Настройки**: Glassmorphic карточки.

### script.js
- `updatePillSlider(container)`: Универсальная функция для позиционирования pill slider по активному элементу.
- Все sliders пересчитываются при: клике на элемент, смене языка (`languageChanged` event), смене темы, загрузке страницы (`requestAnimationFrame`).
- Тема управляется через **glass-dropdown** в шапке и в настройках — синхронизированы.
- `applyPerfMode(enabled)`: включает/выключает атрибут `data-no-blur` на `<html>`, сохраняет состояние в `localStorage`.
- `applyPattern(pattern)`: устанавливает атрибут `data-pattern` на `<html>` для выбора орнамента фона.

### style.css (модульная структура)
Главный `style.css` импортирует 5 модулей через `@import` с использованием системы **кэш-бастинга** (параметры `?v=...`):
- **`css/variables.css`** — CSS-переменные, `@font-face`, семантические токены.
- **`css/layout.css`** — Reset, глобальное масштабирование, сетка workspace.
- **`css/glass.css`** — Навигация, слайдеры, выпадающие списки тем.
- **`css/components.css`** — Кнопки (увеличена высота до **63px**), поля ввода (синхронизированы с высотой ячеек матрицы — **42px**), матрицы, скроллбары.
- **`css/pages.css`** — Карточки настроек, справки и результатов.
