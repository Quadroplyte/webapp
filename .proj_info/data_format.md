# Формат данных

Проект оперирует набором числовых параметров, описывающих задачу выбора СЗИ.

## Входные данные (Алгоритм и API)
- **`m`** (int): Количество групп активов (ГИА). Это количество строк в матрице `A`.
- **`n`** (int): Количество доступных СЗИ. Это количество столбцов в матрице `A`, а также длина векторов `c` и `d`.
- **`A`** (List[List[float]]): Матрица стоимости `m x n`, где `A[i][j]` — стоимость защиты `i`-й группы активов при помощи `j`-го СЗИ.
- **`b`** (List[float]): Вектор доступного бюджета для каждой группы активов (длина `m`).
- **`c`** (List[float]): Оценка времени взлома каждой СЗИ (критерий 1).
- **`d`** (List[float]): Время реакции (скорость обнаружения) каждой СЗИ (критерий 2).
- **`lambda` (λ)** (float): Весовой коэффициент от 0 до 1. При λ=1 учитывается только `c`, при λ=0 — только `d`.

## Примеры файлов данных
- **`input.txt`**: Небольшой пример задачи для быстрой проверки.
- **`test_data_10_10.txt`**: Пример 10x10 с последовательными бюджетами.
- **`test_data_10_10_random_b.txt`**: Пример 10x10 с произвольным порядком бюджетов.
- **`test_data_20_gia.txt`**: Большой пример на 20 групп активов с убывающими лимитами бюджета.
- **`test_data_40.txt`**: Расширенный набор данных (40 СЗИ) для тестирования производительности.

## Логика загрузки (на фронтенде)
Фронтенд считывает текстовый файл, парсит его построчно и преобразует в JSON-объект `SolveRequest`, который затем отправляется на бэкенд.
