# Сборка и деплой

Проект поддерживает два основных типа развертывания: серверный и клиентский (EXE).

## Railway (Серверный деплой)
Конфигурационные файлы:
- `railway.toml`: Описывает шаги сборки и запуска на платформе Railway. Устанавливает системные пакеты (например, `coinor-cbc` для `pulp`).
- `Procfile`: Стандартный файл процессов для запуска FastAPI через `uvicorn`.
- `requirements-server.txt`: Облегченный список зависимостей для сборки в облаке.

## PyInstaller (Windows EXE)
Для превращения Python-скрипта в автономный Windows-файл:
- `main_window.spec`: Файл спецификации PyInstaller. Настроен на сборку одного файла (`onefile`).
- Включает всю папку `static/` внутрь исполняемого файла.
- Скрипт `build_exe.bat` автоматизирует вызов PyInstaller.
- В приложении реализована логика динамического извлечения статических файлов (`sys._MEIPASS`) при работе в "замороженном" (frozen) режиме.

### Команда для ручной сборки:
```bash
pyinstaller main_window.spec
```
Результат будет находиться в папке `dist/`.
