# Общий контекст проекта SZI/IST Knapsack Solver

Данный файл содержит краткую сводку для LLM, чтобы быстро войти в курс дела.

## Назначение
Решение задачи выбора набора СЗИ (средств защиты информации, англ. IST — Information Security Tools) для защиты группы информационных активов (ГИА, англ. AG — Asset Groups) при заданных бюджетных ограничениях.

## Основные сущности
- **СЗИ / ISS (Information Security Solution):** в русской локализации используется термин **СЗИ**, в английской — **ISS**. Имеет два параметра (время взлома `c` и время реакции `d`).
- **ГИА / AG (Group of Information Assets):** в русской локализации — **ГИА**, в английской — **Asset Group (AG)**. Набор активов с общим бюджетом.

## Система тем
Интерфейс поддерживает переключение тем (Темная/Светлая) через CSS-переменные. Все цвета инкапсулированы в семантические токены (`--color-bg-base`, `--color-accent-main` и т.д.), что позволяет расширять систему новыми темами без изменения логики компонентов.

## Краткое описание файлов
- `app.py`: FastAPI бэкенд (порт 8000). Маркирует `/static` и предоставляет `/api/solve`.
- `solver.py`: Алгоритм решения. Использует `pulp` (линейное программирование).
- `main_window.py`: Лаунчер для десктоп-версии (pywebview).
- `static/index.html`: Главный интерфейс (Оптимизация, Документация, Настройки).
- `static/lang.js`: Система локализации (RU/EN). Содержит все строки перевода и логику переключения.
- `static/script.js`: Логика фронтенда (Fetch API, построение таблиц, динамическое обновление результатов при смене языка).
- `static/style.css`: Описание стилей, темная/светлая темы, кастомные переключатели (язык, тема).
- `requirements.txt`: Список всех зависимостей.
- `railway.toml`, `Procfile`: Конфиги для облачного деплоя.
- `build_exe.bat`, `main_window.spec`: Сборка под Windows (Portable EXE).

## Логика решения
Алгоритм является гибридным и итерационным:
1. Решает базовую задачу о рюкзаке ($x_0$) через PuLP.
2. Генерирует $n-1$ вариантов ($x_1...x_{n-1}$), последовательно отбрасывая столбцы в исходном порядке.
3. Выбирает лучшее решение на основе комбинированного критерия $F(x)$.

Вход (JSON): `m, n, lam, A, b, c, d`
Выход (JSON): `vector_x, optimal_f, recommended_szi, all_solutions`
