# Общий контекст проекта SZI/IST Knapsack Solver

Данный файл содержит краткую сводку для LLM, чтобы быстро войти в курс дела.

## Назначение
Решение задачи выбора набора СЗИ (средств защиты информации, англ. IST — Information Security Tools) для защиты группы информационных активов (ГИА, англ. AG — Asset Groups) при заданных финансовых ограничениях.

## Основные сущности
- **СЗИ / ISS (Information Security Solution):** в русской локализации используется термин **СЗИ**, в английской — **ISS**. Имеет два параметра (время обхода `c` и минимальное время реагирования `d`).
- **ГИА / AG (Group of Information Assets):** в русской локализации — **ГИА**, в английской — **Asset Group (AG)**. Набор активов с общим финансовым ограничением.

## Система тем и визуальный стиль
Интерфейс поддерживает переключение тем (Тёмная/Светлая) через CSS-переменные и атрибут `data-theme` на `<html>`.
- **Глобальное масштабирование**: Интерфейс увеличен на **30%** (body: 17px) для улучшения читаемости на современных мониторах.
- **Индивидуальная подгонка**: Основной заголовок в русской версии уменьшен до **32px** (против 42px в англ.) для эстетичного отображения.
- **Семантические токены**: Все цвета инкапсулированы в токены (`--color-bg-base`, `--color-accent-main` и т.д.).

### Глобальный фон
- Фон задан через CSS-переменную `--app-gradient` как `radial-gradient` от нижнего правого угла.
- **Светлая тема**: голубой → нежно-розовый → серый (`#BFDBFE` → `#FCE7F3` → `#D1D3D9`).
- **Тёмная тема**: тёмно-янтарный → тёмно-синий → почти чёрный (`#2C200B` → `#101D30` → `#141414`).
- Поверх градиента наложен орнамент фона, выбираемый в настройках:
  - **Точки** — точечная сетка через `radial-gradient`.
  - **Плюсики** — сетка SVG `+` символов (38×38px).
  - **Крестики** — сетка SVG `×` символов (38×38px, диагональные линии).
  - **Нет** — чистый градиент без орнамента.
- Паттерн выбирается через атрибут `data-pattern` на `<html>` (`dots` / `plus` / `cross` / `none`).
- Фон применён к `body` с `background-attachment: fixed`.

### Акцентные цвета и Габариты
- **Светлая тема**: акцент убран — все кнопки используют нейтральный `--panel-bg` (белый). Выделение лучшей карточки — розово-голубое звёздное свечение (`#F472B6` + `#BFDBFE`).
- **Тёмная тема**: акцент убран — все кнопки используют тёмную поверхность `#222222`. Выделение лучшей карточки — золотое свечение (`#FACC15`).
- **Габариты элементов**:
  - Все основные кнопки действий имеют увеличенную высоту **63px**.
  - Поля ввода параметров (m, n, λ) синхронизированы по высоте с ячейками матриц (**42px**).

### Glassmorphism (стеклянные элементы)
Проект использует glassmorphism как основной стиль UI-компонентов:
- **Glass Pill Nav** (`.glass-pill-nav`): стеклянные rounded-pill контейнеры, `backdrop-filter: blur(24px)`, полупрозрачные градиентные фоны.
- **Convex Styling**: Модальные окна и блоки кода используют эффект «выпуклого» стекла. Теневой профиль настроен на создание объема без использования `inset` теней.
- **3D Glass Title**: Основной заголовок имитирует объемное стекло. В светлой теме — это затемненное прозрачное стекло с белыми бликами на ребрах и мягким парящим ореолом тени.
- **Settings/Docs Glass Cards** (`.settings-glass-card`): карточки с frosted glass эффектом, `backdrop-filter: blur(24px)`, `border-radius: 16px`.

### Страница настроек
- Центрированный layout (`max-width: 480px`).
- Карточка **«Язык интерфейса»** с glass-pill selector.
- Карточка **«Тема оформления»** с glass-dropdown (выбор из списка: Светлая / Тёмная).
- Карточка **«Режим производительности»** с glass-toggle: при включении атрибут `data-no-blur` на `<html>` отключает все `backdrop-filter` и заменяет прозрачные фоны на непрозрачные поверхности из `--color-bg-surface`. Состояние сохраняется в `localStorage`.
- Карточка **«Орнамент фона»** с glass-pill selector (Точки / Плюсики / Крестики / Нет).
- Карточка **«Расположение навигации»** (Статично / По центру / Справа).
- Hover-эффекты на карточках отключены.

### Адаптивность (Mobile)
На экранах `< 1024px` активируется мобильная версия:
- Колонки на странице Оптимизации (`.opt-left-col`, `.opt-right-col`) выстраиваются в один столбик.
- Блоки кандидатов получают собственный скролл поверх страницы.
- Навигационный бар перемещается в самый низ (с учетом отступов `padding-bottom` для `workspace`).
- Добавлены раздельные span-классы `.d-text` и `.m-text` для смены длинных надписей (Справа -> ПК / Внизу -> Тел).

### Страница справки
- Аналогичный центрированный layout (`max-width: 960px`).
- Те же `.settings-glass-card` карточки, но шире и со статичным hover.
- Параметры ввода оформлены через `.docs-params-list` с row-based таблицей.
- Блок пояснений к примеру файла — `.docs-example-guide`.

## Краткое описание файлов
- `app.py`: FastAPI бэкенд (порт 8000). Маркирует `/static` и предоставляет `/api/solve`.
- `solver.py`: Алгоритм решения. Использует `pulp` (линейное программирование).
- `main_window.py`: Лаунчер для десктоп-версии (pywebview).
- `static/index.html`: Главный интерфейс (Оптимизация, Справка, Настройки).
- `static/lang.js`: Система локализации (RU/EN). Содержит все строки перевода и логику переключения.
- `static/script.js`: Логика фронтенда (Fetch API, построение таблиц, pill slider управление, динамическое обновление результатов при смене языка).
- `static/style.css`: Главный CSS-файл — импортирует 5 модулей из `static/css/`.
- `static/css/variables.css`: CSS-переменные, шрифт, семантические токены, dark theme overrides.
- `static/css/layout.css`: Reset, body, app wrapper, header, workspace grid, панели.
- `static/css/glass.css`: Glass pill nav, pill slider, pill items, glass toggle.
- `static/css/components.css`: Кнопки, inputs, params table, матрицы, scrollbar, модалки.
- `static/css/pages.css`: Settings, Docs, Results, кандидаты, glass cards.
- `requirements.txt`: Список всех зависимостей.
- `railway.toml`, `Procfile`: Конфиги для облачного деплоя.
- `build_exe.bat`, `main_window.spec`: Сборка под Windows (Portable EXE).

## Логика решения
Алгоритм является гибридным и итерационным:
1. Решает базовую задачу о рюкзаке ($x_0$) через PuLP.
2. Генерирует $n-1$ вариантов ($x_1...x_{n-1}$), последовательно отбрасывая столбцы в исходном порядке.
3. Выбирает лучшее решение на основе комбинированного критерия $F(x)$.

Вход (JSON): `m, n, lam, A, b, c, d`
Выход (JSON): `vector_x, optimal_f, recommended_szi, all_solutions`
